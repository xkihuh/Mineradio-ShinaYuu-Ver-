# Automatic Update Setup

ShinaYuu Music 1.1.7 checks for a newer public release about ten seconds after startup and every six hours afterward.

## 1. Configure the release repository

Open `package.json` and set:

```json
"mineradio": {
  "update": {
    "provider": "github",
    "owner": "YOUR_GITHUB_ACCOUNT_OR_ORGANIZATION",
    "repo": "YOUR_PUBLIC_RELEASE_REPOSITORY",
    "preview": false,
    "preferMirrors": false,
    "mirrors": [],
    "checkDelayMs": 10000,
    "checkIntervalMs": 21600000,
    "autoPrompt": true
  }
}
```

The same values can be provided at runtime through `MINERADIO_UPDATE_OWNER` and `MINERADIO_UPDATE_REPO`.

## 2. Build one official Windows release

Run:

```powershell
npm run build:win
```

The `dist` folder will contain the installer, its checksum, and `latest.yml`. A blockmap may also be generated by electron-builder.

## 3. Publish one matching GitHub Release

Create a public release with tag:

```text
v1.1.7
```

Upload files from the same build:

- `ShinaYuu-Music-1.1.7-Setup.exe`
- `ShinaYuu-Music-1.1.7-Setup.exe.blockmap` when present
- `latest.yml`

Do not mix an installer from one build with metadata from another build.

## User experience

- The app checks in the background without blocking Spotify or YouTube.
- A newer version opens the in-app Liquid Glass update panel once.
- The user can download with visible progress.
- **Install now** closes the app and starts the installer.
- **Later** keeps using the current version. If the installer is already downloaded, it is scheduled to launch when the user exits the app.

## Rounded-window fallback

The normal Windows window uses the original transparent 30 px rounded shell. For a machine that cannot render transparent Electron windows correctly, launch once with:

```text
ShinaYuuMusic.exe --safe-opaque-window
```

This fallback keeps the startup compatibility behavior but uses an opaque native surface.
